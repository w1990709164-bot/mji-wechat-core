# M叽低成本主动陪伴策略 v1

## 核心原则

主动陪伴不直接照搬 Cyberboss 的“随机唤醒后再让模型决定是否沉默”。M叽是多用户产品，必须先在本地完成筛选，只有合格候选才允许调用模型。

处理链路：

```text
本地候选筛选
→ 用户每日上限检查
→ 全局每日预算检查
→ 个人免打扰与间隔检查
→ 余额与账号状态检查
→ 相关性评分与随机门控
→ 调用模型生成一条主动消息
```

本地规则检查不会调用模型，也不会扣除额度。

## 用户个人设置

每位用户都可以在微信中自行设置每天次数、主动间隔和免打扰时间：

```text
主动消息
主动消息 0
主动消息 20
主动间隔 90分钟
主动间隔 2小时
主动间隔 1天
免打扰 23:30-08:00
关闭免打扰
开启免打扰
关闭主动
开启主动
```

规则：

- 默认每天最多 1 次
- 每天次数不设产品层面的固定上限，由用户填写非负整数
- 0 表示彻底关闭主动消息
- 主动间隔必须大于 0，支持分钟、小时和天
- 免打扰支持跨午夜时间段
- 关闭免打扰后，全天都可进入本地候选筛选
- 用户设置的是允许上限，不代表每天一定发满
- 用户个人设置仍不能突破管理员全局每日预算

## 旧数据库兼容修复

历史数据库中的 `wake_preferences` 约束曾限制主动间隔最少 15 分钟、每日次数最多 50 次。更新本功能后，停止机器人并执行一次：

```powershell
npm run db:fix-proactive
```

该命令只更新主动偏好表的检查约束，不修改用户、余额、订单、聊天或人设数据。它可以重复执行。

## 默认成本保护

- 全局每天最多 20 次主动模型调用
- 只考虑最近 7 天活跃的用户
- 用户最后发言 2 小时内不触发
- 新用户默认两次主动消息至少间隔 8 小时
- 新用户默认免打扰时间 23:00—08:00
- 用户可自行修改个人间隔和免打扰时间
- 用户暂停、管理员暂停、封禁时不触发
- 可用余额不足 10 额度时不触发
- 每次轮询最多放行 1 位用户
- 调用失败自动释放预留额度

模型正式生成主动消息时，按一次正常回复扣 10 额度。额度流水会标记为“主动消息消费”。

## 管理员环境变量

```dotenv
MJI_PROACTIVE_ENABLED=true
MJI_PROACTIVE_GLOBAL_DAILY_LIMIT=20
MJI_PROACTIVE_POLL_MS=60000
MJI_PROACTIVE_MIN_INACTIVITY_MINUTES=120
MJI_PROACTIVE_ACTIVE_WINDOW_DAYS=7
MJI_PROACTIVE_MIN_INTERVAL_MINUTES=240
MJI_PROACTIVE_MAX_INTERVAL_MINUTES=720
MJI_PROACTIVE_MINIMUM_GAP_MINUTES=480
```

管理员可以关闭全局主动消息或调整新用户默认值。已由用户主动保存的个人间隔和免打扰时间会优先执行；所有用户仍受全局每日预算限制。

## 主动内容要求

- 禁止只发“在吗”“干嘛呢”“怎么不回”
- 禁止提及系统、监控、预算、额度、触发器
- 优先跟进未完成的话题、承诺、情绪和近期生活细节
- 可以分享角色自己的小事或自然想起的内容
- 一次只发一条短微信消息
- 避免审问式连续提问
- 必须遵守用户人设、关系阶段和边界
