# 新用户如何添加 M叽

M叽不是管理员的私人微信号，也不是另一个普通个人微信账号。

当前项目通过微信 iLink 的机器人通道运行。执行登录命令时，程序向 iLink 获取一个临时机器人二维码；管理员用微信扫描并确认后，系统返回独立的机器人账号、机器人令牌和 iLink 用户标识，M叽才会出现在微信中并开始收发消息。

## 两种二维码不要混淆

### 1. 程序登录二维码

运行：

```powershell
npm run login
```

终端显示的二维码来自机器人登录接口，用于连接或重新绑定 M叽机器人。

它具有以下特点：

- 不是管理员的私人微信二维码；
- 不是普通微信好友二维码；
- 是临时连接二维码；
- 大约 5 分钟后失效；
- 扫描并确认后会生成或连接一个独立的 `.bot` 机器人账号；
- 不能把它当作长期公开的新用户邀请二维码；
- 不应让每个新用户都重新执行登录流程，否则可能连接出新的机器人账号或改变当前账号状态。

### 2. M叽机器人的用户入口

新用户需要添加的是已经连接成功的 M叽机器人本身，而不是管理员的微信号。

优先检查已经添加好的 M叽聊天或资料页中是否有：

- 推荐给朋友；
- 分享名片；
- 机器人二维码；
- 邀请用户；
- 分享到群聊。

有这些入口时，应把 M叽机器人的名片或二维码分享给新用户。新用户打开后给 M叽发送第一条消息，系统才会自动为他建立独立账户并赠送试用额度。

如果当前 M叽资料页没有任何分享、名片或二维码入口，说明现有代码只完成了机器人登录和消息收发，还没有实现可重复使用的公开邀请二维码。此时不能用管理员私人微信二维码代替，也不要反复运行 `npm run login` 给每个用户扫码。

## 新用户首次消息后的系统流程

新用户成功进入同一个 M叽机器人会话并发送第一条消息后，系统会：

1. 根据发送者的 iLink 用户标识建立独立用户档案；
2. 建立独立聊天记录；
3. 建立长期记忆空间；
4. 建立余额账户；
5. 自动赠送 100 试用额度；
6. 第一次成功回复扣除 10 额度，通常剩余 90。

## 需要保持运行

用户发消息时，运行 M叽的电脑或服务器必须保持：

```powershell
npm start
```

程序停止时，机器人入口仍可能存在，但 AI 不会处理新消息。

## 注意

- M叽是独立 iLink 机器人账号，不是私人微信账号；
- 登录二维码和用户邀请二维码不是同一个东西；
- 同一个 iLink 用户只会建立一个 M叽账户；
- 管理员余额窗口只用于管理员自己，不是普通用户的入口；
- 不要公开数据库、API 密钥、机器人令牌或管理员窗口地址。
