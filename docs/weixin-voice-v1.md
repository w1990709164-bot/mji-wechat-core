# M叽微信语音与多用户音色 v1

## 当前结论

硅基流动克隆音色可以正常生成 MP3，微信媒体上传也可以完成，`sendmessage` 接口会返回成功。

但实际测试中，微信客户端没有收到主动发送的 `voice_item`。因此接口返回成功不能视为客户端送达成功，当前版本不能宣称已经实现真正的微信语音气泡。

## 已验证

- 用户发送微信语音后，M叽可以读取转写并理解
- 克隆音色 MP3 可以正常生成
- MP3 使用 32000 Hz 可正常生成
- 微信媒体上传成功
- `sendmessage` 返回成功
- 微信客户端未收到语音气泡
- 不扣 M叽用户额度
- 临时音频会自动清理

## 安全处理

实验性 `voice_item` 真实测试默认停止，避免继续消耗 TTS 余额。只有协议诊断时才允许显式强制执行。

## 后续路径

1. 继续使用官方 iLink 通道：把 MP3 作为音频文件附件发送，可送达，但不是绿色语音气泡。
2. 要真正的微信语音气泡：更换为能够操作真实微信客户端的其他桥接传输层，并单独评估稳定性与账号风险。

## 多用户音色规则

1. 用户对当前角色的专属音色
2. 当前角色默认音色
3. 系统备用音色

音色配置同时绑定用户、角色、机器人账号和 TTS 供应商。
