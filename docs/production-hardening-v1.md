# M叽生产环境收尾 v1

## 目标

在继续增加陪伴功能之前，先把现有微信版 M叽的正式运行入口、真实发送测试和自动检查收紧，降低误操作、误扣费和后台展示不一致的风险。

## 本 PR 范围

### 1. 正式后台接入

- `npm run admin:web` 启动增强后的事件与角色承诺后台
- `npm run admin:events` 直接启动增强后的事件与角色承诺后台
- 不再要求管理员记住并手动运行临时 v2 脚本
- 保留现有端口和用户管理后台入口

### 2. 真实发送测试安全锁

以下会真实发送微信或真实扣额度的测试命令必须加保护：

- 随机主动消息测试
- 用户事件跟进测试
- 角色承诺兑现测试

保护规则：

- 必须显式设置 `MJI_ALLOW_LIVE_TESTS=1`
- 必须显式指定完整用户 UUID
- 多账号环境禁止自动选择用户
- 执行前打印用户 UUID、微信 ID、机器人账号、当前余额、预计扣费
- 不满足条件时直接拒绝，不创建测试事件
- 测试失败不得误扣费

### 3. 自动检查补全

将以下正式脚本加入 `npm run check`：

- 增强版事件与承诺后台
- 承诺测试用户只读诊断脚本
- 三类真实发送测试准备脚本

## 不在本 PR 范围

- 自动重启和进程守护
- 数据库自动备份
- 外部监控告警
- 隐私协议页面
- 账务日报与运营报表

这些作为下一阶段生产运维 PR 单独实施，避免一次改动过大。
