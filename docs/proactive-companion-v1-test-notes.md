# 主动陪伴测试备注

设置查询和本地筛选不会调用模型。候选通过全部规则并进入生成阶段后，按正常回复预留 10 额度；成功后扣除，失败自动释放。

旧数据库的 `wake_preferences` 表曾限制：

- 主动间隔最少 15 分钟
- 每天主动消息最多 50 次

拉取本分支后，需要先停止机器人并执行一次：

```powershell
npm run db:fix-proactive
```

该命令只放宽主动消息偏好表的检查约束，不修改用户、余额、订单、聊天或人设数据。执行成功后重新启动机器人，再依次发送：

```text
主动间隔 1分钟
主动消息
```

设置页应显示“主动间隔：1 分钟”，不再出现“自助功能暂时无法使用”。

## PostgreSQL 瞬时断线

连接池现在会监听空闲连接断开事件并自动移除坏连接。事务中的连接如果已经断开，会跳过无效回滚并以错误参数释放，使连接池销毁该客户端。主动轮询本次失败后，下一轮会重新获取连接，不应再出现未处理的 `uncaught exception`，也不应连续出现 `client is not queryable`。
