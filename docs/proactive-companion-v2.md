# M叽事件驱动主动陪伴 v2

## 第一阶段目标

把主动消息从“时间到了随机找话题”升级为“记得用户明确提到的未来事件，并在合适时间自然跟进”。

第一阶段只处理**明确、可落地、带有时间线索的事件**，例如：

- 明天下午去医院
- 后天考试
- 晚上和朋友吃饭
- 一会儿面试
- 周五出差
- 下班后去接猫

暂不对模糊内容做强推断，例如“最近可能很忙”“以后想换工作”。模糊内容可以继续作为普通长期记忆，但不直接生成事件触发任务。

## 核心链路

```text
用户发送消息
→ 本地提取明确未来事件
→ 保存事件候选
→ 计算建议跟进时间
→ 到期后再次检查用户状态、免打扰、余额和全局预算
→ 生成一条自然跟进消息
→ 成功发送后标记完成
```

事件提取本身不调用模型、不扣额度。只有事件到期并通过全部本地规则后，才调用模型生成主动消息，按正常主动回复扣 10 额度。

## 数据模型

新增 `proactive_events` 表，独立保存可执行的主动事件，不把调度状态塞进普通长期记忆 JSON。

建议字段：

- `tenant_id`
- `user_id`
- `user_character_id`
- `conversation_id`
- `event_type`
- `title`
- `description`
- `event_at`
- `follow_up_at`
- `status`
- `source_message_id`
- `dedupe_key`
- `metadata`
- `attempt_count`
- `last_attempt_at`
- `completed_at`
- `created_at`
- `updated_at`

状态：

- `pending`：已识别，等待触发
- `queued`：已进入主动消息队列
- `sent`：已成功跟进
- `dismissed`：被用户取消或关闭
- `expired`：超过有效期，不再打扰
- `failed`：连续失败后停止

## 第一阶段本地识别范围

优先支持中文明确时间表达：

- 今天、今晚、明天、明早、明晚、后天
- 一会儿、等会儿、过一小时、两个小时后
- 周一至周日、本周五、下周三
- 上午、中午、下午、晚上、凌晨
- `HH:mm`、`H点`、`H点半`

优先识别事件动作：

- 考试、面试、开会、上班、下班
- 去医院、看医生、复诊、体检
- 出差、旅行、坐车、坐飞机
- 聚餐、约会、见朋友、看电影
- 接猫、接人、取快递、办事

第一阶段宁可少识别，也不误识别。

## 跟进时间规则

- 有明确结束时间：结束后 30—90 分钟跟进
- 只有开始时间：根据事件类型使用保守时长估计
- 只有“明天/后天”没有具体时刻：默认当天 19:00 后跟进
- 跨入个人免打扰：顺延至免打扰结束后
- 用户在事件结束后已经主动聊过该事件：不重复追问
- 每个事件最多主动跟进一次

## 内容要求

主动消息必须：

- 明确承接该事件，但不要像系统提醒
- 符合当前角色人设和关系阶段
- 一次只问一个重点
- 不说“根据记录”“系统显示”“你之前提到”
- 避免固定模板和客服语气

示例：

用户：`明天下午要去医院复诊。`

不推荐：`你今天复诊结果怎么样？`

推荐：`刚想起来你今天去复诊了。回来了吗，医生怎么说？`

## 成本与防打扰

- 事件提取免费
- 事件存储免费
- 到期筛选免费
- 真正生成并发送主动消息才扣 10 额度
- 仍受用户个人每日上限、主动间隔、免打扰和管理员全局预算限制
- 暂停、封禁、余额不足时不触发
- 同一源消息和同一时间窗口必须去重
- 医疗、财务、家庭冲突等敏感事件只做温和跟进，不作判断或建议

## 实施顺序

### 阶段 A：事件基础设施

- [ ] 新增 `proactive_events` 表和迁移
- [ ] 新增 repository
- [ ] 支持创建、去重、到期查询、完成和取消
- [ ] 加入 RLS 和索引

### 阶段 B：本地事件提取

- [ ] 提取相对日期和具体时间
- [ ] 提取常见事件动作
- [ ] 计算 `event_at` 和 `follow_up_at`
- [ ] 低置信度内容不入库
- [ ] 自动化测试覆盖跨午夜、下周、时间缺失和误识别

### 阶段 C：主动发送接入

- [ ] 到期事件优先于随机主动消息
- [ ] 复用 v1 的余额、免打扰、上限和失败释放机制
- [ ] 成功后标记 `sent`
- [ ] 失败重试和最终停止
- [ ] 后台展示触发原因

### 阶段 D：回归

- [ ] 明天事件能正确入库
- [ ] 到期后实际发送
- [ ] 用户已主动聊过时不重复发送
- [ ] 关闭主动、免打扰、余额不足能拦截
- [ ] 多用户事件完全隔离
- [ ] 普通聊天、记忆、人设、充值和后台不受影响

## 后续阶段

第一阶段稳定后再增加：

1. 角色承诺追踪：角色说“晚上提醒你”后真正执行
2. 未完成话题追踪：没有明确时间，但需要稍后回访
3. 生活规律建模：上下班、午睡、周期性安排
4. 角色情境轨迹：角色自己有工作、休息和生活事件
