# 可视化微信扫码接入

启动本机管理员页面：

```powershell
npm run admin:onboard
```

默认打开：

```text
http://127.0.0.1:8789/
```

页面功能：

- 点击生成一张 iLink 临时二维码；
- 显示等待扫码、已扫码待确认、接入成功、失败等状态；
- 二维码约 5 分钟过期后自动刷新；
- 每次接入最多生成 3 张临时二维码；
- 微信确认后自动保存新的机器人账号；
- 页面不显示或返回机器人令牌；
- 已运行的多账号监督器会自动发现新账号并启动对应 worker；
- 页面展示当前已保存的机器人账号列表。

## 正确接入方式

每位新用户都必须扫描属于自己的临时二维码：

1. 管理员点击“生成二维码”；
2. 新用户使用准备接入 M叽的微信账号扫码；
3. 新用户在微信中确认；
4. 页面显示接入成功；
5. 新账号自动保存并进入多账号运行体系。

同一个微信账号再次扫码属于重新绑定。系统会清理该微信账号之前遗留的旧机器人账号文件，避免重复 worker。

## 安全边界

- 管理页面只监听 `127.0.0.1`；
- 不要通过端口映射、隧道或反向代理公开；
- 临时二维码只用于一次接入，不是永久好友二维码；
- 浏览器只收到二维码 SVG、状态和脱敏后的账号信息；
- `bot_token` 只在服务器内存和本地账号文件中处理；
- 关闭 PowerShell 窗口后，管理页面立即停止。
