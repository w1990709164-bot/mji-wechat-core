# M叽角色承诺追踪 v2.1

## 目标

让角色说出的明确承诺真正执行，而不是只在当前回复中制造陪伴感。

示例：

- 晚点我再来找你。
- 两个小时后陪你聊。
- 明早叫你起床。
- 下班后我来陪你。
- 等你面试结束，我来问你结果。

## 核心原则

1. **先落库，后提取**：只分析已经成功保存的角色回复。
2. **本地保守识别**：提取和保存不调用模型、不扣额度。
3. **明确承诺才记录**：必须同时具有角色第一人称承诺动作，以及未来时间或可绑定的用户事件。
4. **执行才扣费**：真正生成并发送承诺兑现消息时，沿用现有主动消息消费规则，成功发送扣 10 额度，失败释放预留额度。
5. **不绕过用户设置**：继续遵守主动消息开关、个人每日上限、全局预算、最低间隔、免打扰和余额门槛。
6. **禁止自循环**：事件跟进、随机主动消息和承诺兑现消息本身不得再次生成新承诺。
7. **宁可漏识别，不误识别**：含“可能、也许、尽量、看情况、有空的话”等不确定表达时不保存。

## 数据设计

第一阶段复用 `proactive_events`，不新增独立业务表。

角色承诺使用：

- `event_type = character_promise`
- `source_message_id` 指向角色的 assistant 消息
- `event_at` 表示承诺约定的时间或绑定用户事件的结束时间
- `follow_up_at` 表示允许执行的最早时间
- `metadata.triggerKind = character_promise`
- `metadata.promiseAction` 表示承诺动作
- `metadata.promiseText` 保存角色原话
- `metadata.linkedProactiveEventId` 保存绑定的用户事件（如面试）
- `metadata.sourceRole = assistant`

## 第一阶段承诺动作

- `return_chat`：晚点再来找用户、回来继续聊
- `accompany`：下班后或约定时间陪用户聊天
- `wake_up`：约定时间叫用户起床
- `ask_result`：在用户事件结束后询问结果
- `remind`：角色明确承诺之后提醒用户

## 时间识别

复用 v2 已有时区和中文时间解析能力，并补充：

- 晚点、等会儿、稍后
- 明早、明晚
- 两个小时后
- 下班后、午休后、睡醒后（仅在用户已有生活时间资料时解析）
- 等你面试/考试/复诊结束（绑定最近的匹配用户事件）

不具备可靠时间时不自动安排。

## 捕获位置

角色正常回复完成后会写入：

- `direction = outbound`
- `role = assistant`

聊天仓库在消息落库成功后运行承诺提取器。若提取或保存失败，只记录日志，不影响消息发送和普通聊天。

以下来源必须跳过：

- 随机主动消息
- 用户事件跟进消息
- 承诺兑现消息
- 系统提示或本地命令回复
- 角色设定、历史记录和引用文本中的承诺句

## 执行方式

到期后进入现有事件优先主动队列。生成提示根据动作区分：

- `return_chat`：自然回来开启话题，不假装刚收到用户新消息
- `accompany`：兑现陪伴，不客服化
- `wake_up`：简短自然地叫醒用户
- `ask_result`：只问一个聚焦问题，不预设结果
- `remind`：自然提醒，不暴露系统任务

## 去重

同一用户、同一角色、同一承诺动作、同一时间段、同一规范化承诺文本，只保存一条。

角色重复输出同一句、消息重放、程序重启均不得产生重复承诺。

## 第一阶段不做

- 没有时间信息的长期口头承诺
- 复杂条件树和多阶段计划
- 用模型扫描全部历史承诺
- 角色自行修改用户的主动消息设置
- 绕过免打扰强制兑现

## 实施顺序

1. 新增承诺本地提取器和测试
2. 在角色消息落库后捕获承诺
3. 支持绑定最近的用户事件
4. 扩展事件发送提示，按承诺动作兑现
5. 后台显示“角色承诺”触发来源和动作
6. 测试免打扰、余额、去重、多用户隔离和禁止自循环
7. 完成最终回归后合并

## 验收用例

### 应识别

- 晚点我再来找你。
- 两个小时后我来陪你。
- 明早七点我叫你起床。
- 等你面试结束，我来问你结果。

### 不应识别

- 有空的话我再找你。
- 我可能晚点来。
- 我尽量明早叫你。
- 他说明天会来找你。
- 用户历史中写着“我晚点找你”。
- 当前消息是事件跟进或随机主动消息。

### 执行验证

- 保存承诺时不调用模型、不扣额度
- 到期发送成功时只扣 10 额度
- 失败不扣费并按现有规则重试
- 用户关闭主动消息时不执行
- 免打扰期间不执行
- 用户主动聊过后，根据承诺类型决定取消或继续：`return_chat` 可取消，`wake_up/remind` 保留
- 后台能够看到承诺原话、动作、约定时间、状态和扣费结果
