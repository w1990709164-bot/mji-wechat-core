# 新用户自动赠送试用额度

M叽微信版会在微信用户第一次发消息、系统首次为该用户建档时，自动赠送试用额度。

默认赠送：

```env
MJI_NEW_USER_TRIAL_CREDITS=30
```

当前每次成功回复扣除 10 额度，因此默认 30 额度可体验约 3 次成功回复。

## 修改赠送额度

在本机 `.env` 中加入或修改：

```env
MJI_NEW_USER_TRIAL_CREDITS=50
```

保存后重启程序即可。数值可使用小数。

## 关闭自动赠送

```env
MJI_NEW_USER_TRIAL_CREDITS=0
```

## 规则

- 只对功能启用后新建档的用户赠送；
- 已存在的老用户不会被自动补发；
- 每个用户只赠送一次；
- 赠送记录会写入 `wallet_transactions`，类型为 `topup`；
- 流水备注为“新用户试用额度”；
- 使用固定唯一流水键，即使程序中途重启或重复处理，也不会重复赠送；
- 若赠送成功但用户资料标记更新失败，下次消息会继续检查，数据库仍会阻止重复充值。

## 测试

使用一个从未给该微信账号发过消息的新微信用户发送第一条消息，然后在管理员窗口执行：

```powershell
npm run admin:web
```

在用户列表中选择新用户，余额应先增加试用额度，再按成功回复扣除一次费用。

例如默认赠送 30、每次回复扣 10，则首次成功回复后可用余额通常为 20。
