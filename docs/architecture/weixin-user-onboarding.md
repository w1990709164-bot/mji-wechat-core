# 微信用户自动建档流程

M叽微信版在配置 `MJI_DATABASE_URL` 后启用多用户数据库模式。

启动时：

1. 检查 PostgreSQL 连接。
2. 按租户 slug 创建或读取 M叽租户。
3. 将当前登录的微信账号登记到 `channel_accounts`。

收到微信消息时：

1. 使用当前微信账号和发送者 ID 查询 `channel_identities`。
2. 已存在时更新最后活跃时间和最近消息元数据。
3. 不存在时创建 `app_users` 与 `channel_identities`。
4. 将内部 `tenantId`、`userId`、`identityId` 附加到本次消息上下文。
5. 数据库建档失败时停止本次 AI 调用，避免消息进入错误用户的会话或记忆。

当前阶段完成的是用户身份隔离入口。后续会继续接入：

- 每位用户独立的角色关系实例；
- 会话与消息持久化；
- 长期记忆加载与写入；
- 模型调用用量与套餐额度扣减。
