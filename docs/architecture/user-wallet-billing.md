# 用户余额与成功回复扣费

M叽微信版采用“预留 → 成功扣除 → 失败释放”的余额流程。

默认规则：

- 每次准备调用模型前预留 10 额度；
- 模型返回有效回复后正式扣除 10 额度；
- 模型调用失败或取消时释放预留额度；
- 同一次调用通过唯一流水键防止重复扣费；
- 内部重试仍属于同一次调用，不重复向用户收费；
- 余额不足时不调用上游模型，并向用户返回余额不足提示。

环境变量：

```env
MJI_USER_CHARGE_PER_REPLY=10
```

数据库表：

- `user_wallets`：余额、预留额度和钱包状态；
- `wallet_transactions`：充值、预留、扣除、释放、退款和调整流水。

上游 API 成本与用户收费分别记录：

- `usage_records.cost_microunits`：上游实际成本；
- `wallet_transactions`：向用户收取的额度。

两者不可混用。当前约定为：上游每次成本 4.5 额度，用户每次成功回复收取 10 额度。
