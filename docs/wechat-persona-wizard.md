# 微信内人设设置向导

用户在微信中发送：

```text
设置人设
```

也支持：

```text
修改人设
人设设置
角色设置
/persona
```

系统会在调用模型之前拦截这些消息，使用本地固定流程逐项询问：

1. 机器人名字；
2. 机器人如何称呼用户；
3. 身份定位；
4. 性格；
5. 说话方式；
6. 关系阶段；
7. 关系要求；
8. 背景故事；
9. 边界和禁区；
10. 额外专属指令；
11. 确认保存。

## 本地命令

向导过程中可以发送：

```text
上一步
跳过
重新开始
取消
保存
```

## 计费与隐私

- 整个设置流程不调用模型；
- 不产生供应商 API 成本；
- 不扣除用户额度；
- 向导问答不写入普通聊天记录；
- 向导问答不参与长期记忆提取；
- 向导状态按用户保存在数据库，程序重启后仍可继续；
- 保存后，从该用户下一条正常聊天开始加载新的人设；
- 用户之间的人设和向导进度相互隔离。

## 管理员验证

终端会输出：

```text
[mji] local persona wizard ... apiCalled=false creditsCharged=0
```

管理员还可以运行：

```powershell
npm run admin:persona
```

检查并修改每位用户保存的人设。
